## 根因

`app/llm.py` 的两个调用路径均未过滤推理模型（DeepSeek-R1、QwQ 等）输出的 `<think>…</think>` 块：

- `_chat_stream`（行 166）：收集所有 delta.content 后直接 `return "".join(content_parts)`，think 块原样返回给 `generate_sql` / `narrate_result`
- `_stream_chat_chunks`（行 274）：逐 token `yield c`，think 块被分片推送给前端，在洞察总结区域直接显示原始推理过程

## 修复内容

1. `_chat_stream`：在 return 前加 `re.sub(r"<think>.*?</think>", "", result, flags=re.DOTALL).strip()`
2. 新增 `_strip_think_tags(chunks: Iterator[str]) -> Iterator[str]`：流式状态机，处理跨 chunk 分割的 think 标签
3. `stream_insight_summary`：用 `_strip_think_tags(self._stream_chat_chunks(...))` 包装流式调用

## 影响文件

- `app/llm.py`

## 验证

重启服务后，使用 DeepSeek-R1 / QwQ 等推理模型发起洞察分析，UI 不再出现 `<think>` 标签内容。
