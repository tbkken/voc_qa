## 根因

`InsightPipeline.run()` 在 `result.error` 时只推送 `result.answer`（错误文案），
未将 `result.raw_sql`（LLM 生成的原始 SQL）放入 `item_error` payload。
AC-11 的 payload spec 是 `{ index, error }`，未包含 `sql` 字段，属于 Spec 遗漏。

用户看到"不允许多条 SQL 语句"时无法判断 LLM 实际生成了什么，调试困难。

## 修复内容

1. `InsightPipeline.run()`：`item_error` 事件增加 `sql: result.raw_sql` 字段
2. 前端 `item_error` 处理：错误文案下方渲染原始 SQL（`<pre class="insight-item-sql">`，带最大高度和滚动）
3. 新增 CSS `.insight-item-sql`：monospace，深色背景，与错误区域视觉区分
4. 更新 AC-11：payload 增加 sql 字段说明

## 影响文件

- `app/pipeline.py`（`InsightPipeline.run`）
- `web/index.html`（`item_error` handler + CSS）
- `docs/spec/SPEC-001-insight-analysis.md`（AC-11）
