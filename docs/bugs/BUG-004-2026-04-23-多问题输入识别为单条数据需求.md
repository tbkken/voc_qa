## 根因

`_parse_insight_by_rules`（已删除）使用关键词触发列表（"分析以下细节："等）判断是否拆分多条 items。
用户用"？"分隔的多问题输入不含这些关键词，直接落入 `items=[question]` fallback，整段文字作为单条送给 LLM，LLM 生成多语句 SQL，被 sql_guard 以"不允许多条 SQL 语句"拦截。

AC-01 原文说"含列表结构则拆分"，但未定义"列表结构"包含"问号分隔的多问题"，属于 Spec 遗漏。

## 根本原因

用规则判断"有几个需求"是错误的方向——自然语言的表达方式远比规则复杂，LLM 天然具备这种理解能力。

## 修复内容

1. 删除 `_parse_insight_by_rules` 函数
2. `parse_insight_intent` 改为调用 LLM，prompt 要求返回 JSON `{background, items}`
3. LLM 失败时降级为 `items=[question]`，不中断流程
4. 更新 AC-01：明确"LLM 解析，按独立可查询需求拆分"

## 影响文件

- `app/llm.py`（`parse_insight_intent`，删除 `_parse_insight_by_rules`）
- `docs/spec/SPEC-001-insight-analysis.md`（AC-01）
