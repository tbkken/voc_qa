## 根因

两处 LLM 调用均缺乏数据接地约束：

**1. `narrate_result`**
prompt 只说"引用具体数字"，未明确禁止凭训练知识编造。
推理模型在 `<think>` 里"推断"结论后，在正文直接输出结论而不看表格，
导致叙述中的数字与查询结果不符。

**2. `stream_insight_summary`**
只传入前 3 行数据（格式 `r[0]=r[-1]` 仅取首尾列），数据信息严重丢失。
总结 LLM 看不到完整数据，即使 narration 正确也无法准确引用原始数字。

SPEC-001 无任何 AC 要求"叙述必须严格引用查询结果数字"，属于 Spec 遗漏。

## 修复内容

**`narrate_result` prompt 强化：**
- 明确标注"查询结果表格"，要求所有数字直接引用表格原始数值
- 注明当前行数，0 行时只能说"未查询到数据"
- 明确禁止"通常/一般/可能/预计"等推测性表述
- temperature 从 0.3 降至 0.1，减少随机发挥

**`stream_insight_summary` 数据传入改造：**
- 从前 3 行摘要改为传入完整表格（最多 20 行，带列名表头）
- 保留所有列（原来只取首尾列）
- system prompt 明确"所有引用数字必须来自原始数据，严禁凭先验知识编造"

**新增 SPEC-001 AC-25：**
- narration 中所有数字必须来自查询结果，禁止推测性表述

## 影响文件

- `app/llm.py`（`narrate_result`、`stream_insight_summary`）
- `docs/spec/SPEC-001-insight-analysis.md`（AC-25）
