# BUG-001 洞察意图识别失败

**日期**：2026-04-23
**严重程度**：P1（核心功能不可用）
**分流**：A（代码写错）
**解决方式**：full

## 现象

用户输入"对全量数据做一个洞察，分析以下细节：..."，系统返回"未识别为洞察分析请求"。

## 根因

`parse_insight_intent` 依赖真实 LLM 调用来识别意图。真实 LLM 模式下，若 LLM 返回非标准 JSON、超时或调用异常，`except Exception` 静默吞掉错误并返回 `{"is_insight": False}`，导致所有洞察请求被错误拒绝。Mock 模式不受影响（走关键词匹配）。

## 影响范围

- 真实 LLM 模式下，所有洞察分析请求均报"未识别"错误
- Mock 模式正常

## 修复方案

移除 `parse_insight_intent` 的 LLM 调用，改为纯规则解析（不会失败）：
- 含列表触发词（分析以下/分析如下 + 分隔符）→ 拆分多条 items
- 普通单问题 → `items=[question]`（单条洞察，与多条共用同一 pipeline）
- 前端移除意图检测分支，所有问题统一走洞察 pipeline
