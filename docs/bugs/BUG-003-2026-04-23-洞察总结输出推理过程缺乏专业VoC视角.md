## 根因

AC-15 只规定了输出格式（「观点标题」：结论 + 数据佐证），未规定 system prompt 须以专业 VoC 分析专家角色出发，未禁止推理过程输出。

导致 `stream_insight_summary` 的 system prompt 过于宽泛（"你是数据洞察专家"），推理模型在 think 块被过滤后，正文仍倾向于输出分析思路而非直接结论，用户实际看到的是思考过程而非高管报告风格的洞察。

## 修复内容

重写 `stream_insight_summary` system prompt：
- 角色：资深 VoC 分析专家，为管理层撰写洞察报告
- 禁止：引言、推理过程、`<think>` 内容、收尾总结
- 要求：每个结论必须引用具体数值/占比/排名，高管报告风格

同步更新 AC-15，补充专家视角和禁止推理过程要求。

## 影响文件

- `app/llm.py`（`stream_insight_summary`）
- `docs/spec/SPEC-001-insight-analysis.md`（AC-15）

## 验证

重启服务后，洞察总结直接输出「观点标题」格式的结论段落，无引言、无推理过程、无收尾语。
