# Spec: 图表展示优化

**Spec 编号**：SPEC-002
**关联 PRD**：图表展示优化需求（用户口述）
**版本**：v1.0
**状态**：草稿
**创建日期**：2026-04-23

---

## 背景 & 目标

现有 `_suggest_chart`（`app/pipeline.py`）根据查询结果自动选择图表类型，存在三个问题：柱状图方向与惯例相反（横向而非竖向）、缺少饼图导致分布类数据展示不直观、单列纯文本结果仍尝试渲染图表。本次优化修正这三个问题。

**非目标**（本期明确不做）：
- 用户手动切换图表类型
- 散点图、雷达图等其他图表类型
- 图表导出功能

---

## 用户故事与验收标准

### US-01：柱状图改为竖向

**作为** VoC 分析师，
**我希望** TOP 榜类柱状图按惯例竖向展示（X 轴为类目、Y 轴为数值），
**以便** 符合数据可视化阅读习惯，类目标签在 X 轴清晰排列。

**验收标准**：

- [ ] **AC-01**（正常）：Given 查询结果为 2 列以上、首列非时间维度、行数 ≤ 30、末列为数字，When `_suggest_chart` 计算图表类型，Then 返回 `bar`；前端用 ECharts 渲染时 `xAxis.type = 'category'`（类目在 X 轴），`yAxis.type = 'value'`（数值在 Y 轴），柱子竖向生长。

- [ ] **AC-02**（边界）：Given 柱状图类目标签文字较长（超过 6 个汉字），When ECharts 渲染，Then X 轴标签自动旋转 45° 显示，不截断文字，不与相邻标签重叠。

- [ ] **AC-03**（边界）：Given 行数 > 30 的非时间维度多列数据，When `_suggest_chart` 计算，Then 返回 `table`，不渲染柱状图（数据量过大，柱状不可读）。

---

### US-02：新增饼图用于分布类数据

**作为** VoC 分析师，
**我希望** 情感分布、渠道占比、品类分布等"整体与局部"关系的数据用饼图展示，
**以便** 直观看到各部分在整体中的占比，而非逐条比较长短。

**验收标准**：

- [ ] **AC-04**（正常）：Given 查询结果恰好 2 列、行数 ≤ 8、末列值可转为数字（int 或 float），When `_suggest_chart` 计算，Then 返回 `pie`；前端用 ECharts 渲染饼图，`series[0].type = 'pie'`，每个 slice 的 `name` = 首列值、`value` = 末列数字，图例显示在右侧，tooltip 显示名称 + 数值 + 占比百分比。

- [ ] **AC-05**（边界）：Given 查询结果 2 列但行数 > 8，When `_suggest_chart` 计算，Then 返回 `bar`（饼图切片过多，可读性差，降级为柱状图）。

- [ ] **AC-06**（边界）：Given 查询结果 2 列、行数 ≤ 8，但末列值均为非数字字符串，When `_suggest_chart` 计算，Then 返回 `table`（无法量化，不渲染图表）。

- [ ] **AC-07**（边界）：Given 饼图数据中某个 slice 的值为 0，When ECharts 渲染，Then 该 slice 不显示（ECharts 默认行为），图例中仍保留该类目名称。

---

### US-03：单列纯文本结果仅显示表格

**作为** VoC 分析师，
**我希望** 查询返回纯文本内容（如原声评论列表）时不显示图表，
**以便** 避免将文本内容强行渲染成无意义的柱状图，只用表格清晰呈现原文。

**验收标准**：

- [ ] **AC-08**（正常）：Given 查询结果只有 1 列、且该列所有行的值均无法转为数字（即纯字符串，如评论内容、关键词），When `_suggest_chart` 计算，Then 返回 `table`；前端只渲染数据表格，不渲染任何图表区域。

- [ ] **AC-09**（正常）：Given 查询结果 1 列 1 行、且该行值为数字（如"总声量: 12345"），When `_suggest_chart` 计算，Then 返回 `metric`，前端渲染大数字卡片（保持现有行为）。

- [ ] **AC-10**（边界）：Given 查询结果 1 列多行、且所有值均为数字，When `_suggest_chart` 计算，Then 返回 `table`（多行单列数字无合适图表类型，用表格呈现）。

## 数据需求

| 字段名 | 类型 | 说明 | 关联 AC |
|--------|------|------|---------|
| `chart_type` | Enum | `metric` / `bar` / `line` / `pie` / `table` / `none` | AC-01、AC-04、AC-08 |

新增枚举值 `pie`，SSE `item_done` 事件的 `chart_type` 字段可能返回 `pie`。

---

## 接口需求

无新增接口，`item_done` 事件的 `chart_type` 字段新增可能值 `pie`，前端需新增 pie 渲染分支。

---

## 非功能需求

| 类型 | 要求 | 关联 AC |
|------|------|---------|
| 兼容性 | 现有 metric/bar/line/table 渲染逻辑保持不变，只新增 pie 和修改 bar 方向 | AC-01、AC-04 |
| 性能 | `_suggest_chart` 为纯内存计算，执行时间 < 1ms | - |
